# 🏥 基于 OCR 与大模型 NER 的智能病历结构化分析系统

基于深度学习与规则引擎的双驱架构，专为基层非结构化中文门诊病历设计的自动化信息抽取、逻辑推理与全链路快照归档系统。

## 🌟 项目目的与意义

- **规范化基层医疗存储**：针对村镇等基层小诊所纸质病历存储不规范、易丢失的问题，提供低门槛的数字化与结构化解决方案。
- **优化转院对接流程**：将非结构化的手写/打印病历转化为标准结构化 JSON 数据，方便后续向上级医院转院时的病史查阅与底层数据对接。
- **降低数字化门槛**：利用轻量化本地部署方案，解耦底层配置，无需昂贵服务器即可实现商业级电子健康档案（EHR）的管理。

---

## 🧠 系统架构与工作流

系统严格遵循“高内聚、低耦合”的软件工程标准，划分为解耦的三个核心层级：

1. **[感知层] 视觉文本提取 (OCR)**：
   `门诊病历图片` ➔ `RapidOCR 引擎` ➔ `非结构化原始文本`。
   支持倾斜矫正与复杂排版的医学影像文字高精度提取。
2. **[认知层] 自然语言理解与推理 (NLP/NER)**：
   `原始文本` ➔ `Processor 规则引擎截断` ➔ `MacBERT-GlobalPointer-RoPE 抽取` ➔ `逻辑关系组装`。
   采用基于旋转位置编码的全局指针网络，完美解决医疗文本中的实体嵌套问题，并基于语义距离实现高阶临床逻辑推断。
3. **[持久层] 快照溯源持久化 (Storage)**：
   `结构化实体与结论` ➔ `Storage 引擎` ➔ `时间戳全链路快照`。
   生成包含原图备份、原文溯源、段落切分、高阶推断结论的聚合 JSON 电子档案。

---

## ✨ 核心亮点特性 (Advanced Features)

- **防御性规则引擎 (Fail-safe Rule Engine)**：独创基于坐标锚点的病历段落截断法，精准切割“主诉”、“现病史”等模块，内建“短字段防越界”机制，彻底免疫传统正则贪婪匹配带来的崩溃灾难。
- **医疗实体上下文归属 (Contextual Attribution)**：突破传统 NER 局限，提取的每一个医学实体自动绑定所属的病历语境（例如：“心脏病”自动关联至“既往史”）。
- **实体关系交叉推断 (Relation Extraction)**：在单句语境内进行实体碰撞，自动推断并生成“检查结果 (检查+疾病)”、“用药目的 (药物+症状)”、“发病部位 (部位+症状)”及正则提取的时间病程属性。
- **100% 数据驱动架构 (Data-driven)**：业务规则、字典纠错、网络参数、系统设置均抽离至独立的 JSON 配置文件，实现代码与数据的物理隔离。

## 🚀 快速启动与测试

### 1. 环境准备

本项目基于 Python 3.8+ 开发，核心依赖包括深度学习框架与图像识别引擎。请执行以下命令进行一键安装：

```bash
pip install rapidocr_onnxruntime transformers torch pydantic
```

### 2.权重装载与配置确认

模型权重：将微调完成的 PyTorch 权重文件（如`ner_model.pt`）放置于根目录下的 `models/` 文件夹中。

路径检查：打开 `configs/model.json`，确保 `checkpoint_path` 字段指向正确的权重文件路径。

设备设置：若具备 NVIDIA 显卡环境，可在 `configs/global_settings.json` 中将 `device` 设为 `"cuda"` 以加速推理。

### 3.执行全链路管线

将待识别的门诊病历图像（支持常见格式如 R.jpg）放置于项目根目录或 data/ 目录，直接运行主调度器：

```Bash
python main.py
```
*执行过程中，系统将实时输出各阶段处理进度，包括段落切分结果、实体识别明细以及最终的逻辑推断结论。*

## 📂 核心模块深度说明

为了确保系统的可维护性与扩展性，本项目对功能进行了严格的原子化解耦：

### 🛠️ 1. 配置管理中心 (`app/config_manager.py`)

-   **功能描述**：作为系统的“神经中枢”，负责全局配置的聚合与分发。
-   **技术细节**：采用单例模式 (Singleton) 设计，启动时自动扫描 configs/ 目录下所有的 JSON 配置文件，并合并为一个全局配置字典。支持热插拔，修改规则或模型参数无需触动业务逻辑代码。

### 🧠 2. 神经网络架构 (`app/model.py`)

-   **功能描述**：定义了本系统的认知核心算子。
-   **技术细节**：实现了 GlobalPointer (全局指针网络) 架构。与传统的序列标注（CRF/Softmax）不同，该模块通过计算序列中任意两个 Token 作为实体起止点的概率来识别实体。
-   **RoPE 注入**：内置了 旋转位置编码 (Rotary Position Embedding)，通过旋转矩阵捕捉 Token 间的相对位置关系，大幅提升了模型对医学长文本的理解深度。

### 🔎 3. 推理引擎模块 (`app/ner.py`)

-   **功能描述**：负责将预训练语言模型（MacBERT）与自定义解码头进行装配并执行前向计算。
-   **核心逻辑**：集成了 Tokenizer 分词、偏移量映射 (Offset Mapping) 以及打分矩阵解码。该模块输出原始的实体坐标与类别索引。

### 🧰 4. 逻辑处理管家 (`app/processor.py`)

-   **功能描述**：系统中最复杂的业务逻辑层，负责将原始 OCR 文本转化为结构化知识。
-   **核心能力**：
    -   **纠错清洗**：基于配置字典自动修正 OCR 识别中的医学专有名词误记。
    -   **锚点截断**：通过精准搜索病历标题锚点，实现段落的 100% 物理隔离。
    -   **关系推断**：基于语义距离算法，在单句内自动匹配“部位+症状”等关联关系，生成逻辑推断报告。

### 📁 5. 持久化存储引擎 (`app/storage.py`)

-   **功能描述**：负责数据的安全落地与版本化管理。
-   **存储策略**：系统为每位患者分配唯一的 UUID (通用唯一识别码)，并以 V\_时间戳 为单位创建就诊快照。每个快照包内均包含原始影像、中间过程文本及最终结构化 JSON 档案，确保医疗数据的可追溯性（Data Traceability）。
