## **项目目的与意义**

- **规范化基层医疗存储**：针对村镇等基层小诊所纸质病历存储不规范、易丢失的问题，提供低门槛的数字化解决方案。
- **优化转院对接流程**：将非结构化手写病历转化为标准结构化数据，方便后续向上级医院转院时的病史查阅与数据对接。
- **降低数字化门槛**：利用轻量化本地部署方案，无需昂贵服务器即可实现病历的结构化管理。

## **项目结构**

**PART1 感知层 (OCR)**：`手写病历图片` $\xrightarrow{PaddleOCR / OpenCV}$ `非结构化原始文本`。

**PART2 认知层 (NER)**：`原始文本` $\xrightarrow{BERT-GlobalPointer / CRF}$ `CMeEE-V2 结构化实体`。

**PART3 持久层 (DB)**：`结构化实体` $\xrightarrow{FastAPI + MySQL}$ `智能化搜索与病史管理`。

## PART1 感知层 (OCR)

**目标**：识别用户上传图片的手写病例，通过模板对齐方案输出正确的区域化文本。

**实现逻辑**：

1. **模板对齐**：通过预设的标准病历模板，利用 OpenCV 的单应性变换算法，将用户上传的倾斜或形变图片矫正至标准坐标系。
2. **区域裁剪 (ROI)**：按照老师建议，划定病历图像中的文字所属区域（如主诉区、诊断区），确保识别部分不会错误拼合。
3. **文本识别**：对裁剪后的特定区域调用 PaddleOCR 引擎，输出该区域对应的文本流。

## PART2 认知层 (NER)

**目标**：提取输入的病例文本，转化成结构化实体（参考 CMeEE-V2 数据集要求的结构），存储到数据库中对应患者的记录里。

**实现逻辑**：

1. **模型选型**：采用 **BERT + GlobalPointer** 架构，利用全局指针机制解决医疗病历中常见的嵌套实体识别难题。
2. **实体类别**：适配 CMeEE-V2 的 9 大类实体（疾病、症状、药物、身体部位等）。
3. **语义处理**：利用预训练模型的语义理解能力，对 OCR 识别结果进行鲁棒性增强。

## PART3 持久层（DB）

**目标**：提供用户交互界面并实现规范化的本地数据存储。

**实现逻辑**：

1. **用户交互**：提供上传图片功能，识别提取后要求用户在线核对。只有在用户确保无误并确认后，系统才会执行上传存储操作。
2. **树状存储结构**：每个电子病例对应一个患者，拥有唯一的编号索引。采用文件夹分层存储：
   - **一级目录**：患者 ID 与姓名。
   - **二级目录**：就诊记录文件夹（包含单次就诊的原始图片与结构化 `record.json` 文件）。
3. **病历管理**：提供新增、修改、删除电子病历及就诊记录的功能。

**存储结构示例**：

```Plaintext

                ├── source.jpg       # 原始图片
                └── record.json      # 识别出的结构化实体数据
```

#### 1.**[中文医学命名实体识别 V2（CMeEE-V2）](<[中文医疗信息处理评测基准CBLUE_数据集-阿里云天池](https://tianchi.aliyun.com/dataset/95414)>)**

##### 目标描述

    对于医学领域的自然语言文献，例如医学教材、医学百科、临床病例、医学期刊、入院记录、检验报告等，这些文本中蕴含大量医学专业知识和医学术语。将实体识别技术与医学专业领域结合，利用机器读取医学文本，可以显著提高临床科研的效率和质量，并且可服务于下游子任务。要想让机器“读懂”医学数据，核心在于让计算机在大量医学文本中准确的提取出关键信息，这就涉及到了命名实体识别、关系抽取等自然语言处理技术。医学领域中非结构化的文本，都是由中文自然语言句子或句子集合组成。实体抽取是从非结构化医学文本中找出医学实体，如疾病、症状的过程。

_注：相比 CMeEE 原始版本，CMeEE-V2 主要更新是修复了原始数据中的部分标注错误，提升了语料质量。此外实体下标位置由原始的左闭右闭调整为左闭右开标记法。_

##### 说明

    给定schema及句子sentence，对于给定的一组纯医学文本文档，任务的目标是识别并抽取出与医学临床相关的实体，并将他们归类到预先定义好的类别。将医学文本命名实体划分为九大类，包括：疾病(dis)，临床表现(sym)，药物(dru)，医疗设备(equ)，医疗程序(pro)，身体(bod)，医学检验项目(ite)，微生物类(mic)，科室(dep)。标注之前对文章进行自动分词处理，所有的医学实体均已正确切分。

标注规范请参考论文"[面向医学文本处理的医学实体标注规范](https://aclanthology.org/2020.ccl-1.52.pdf)"。

标注的“输入/输出”示例:

1. 输入: 句子 sentence
2. 输出: 句子 sentence 中包含的医学实体位置和实体类型
